### Package B — Apps Script on the signal-system sheet (Task 2)

Why: cuts sheet→Tapeline lag from 5 min (current CSV poll) to 1–3 seconds.
Webhook backend is live (`POST /api/internal/sheet-changed` returns 401 on
wrong secret, confirming `SHEET_WEBHOOK_SECRET` is configured on Fly).

**Step 1 — read the secret (the Apps Script needs to send the same value).**

```powershell
fly ssh console -a tapeline-backend -C 'printenv SHEET_WEBHOOK_SECRET'
```

Copy the value.

**Step 2 — open the "Live Dashboard - Stocks" sheet → Extensions → Apps Script.**

Paste this verbatim into the script editor (replaces the default `Code.gs`
contents):

```javascript
function notifyTapeline() {
  UrlFetchApp.fetch('https://api.tapeline.io/api/internal/sheet-changed', {
    method: 'post',
    contentType: 'application/json',
    payload: JSON.stringify({
      secret: PropertiesService.getScriptProperties().getProperty('TAPELINE_WEBHOOK_SECRET')
    }),
    muteHttpExceptions: true
  });
}
```

**Step 3 — File → Project Settings → Script Properties → Add script property:**

- Name: `TAPELINE_WEBHOOK_SECRET`
- Value: (the value from Step 1)

**Step 4 — Triggers (left sidebar clock icon) → Add Trigger:**

- Function: `notifyTapeline`
- Deployment: `Head`
- Event source: `From spreadsheet`
- Event type: `On change`
- Save (Google will prompt for permissions — approve)

**Step 5 — verify.** Edit any cell on the sheet, then in PowerShell:

```powershell
fly logs -a tapeline-backend | Select-String "internal.sheet-changed"
```

You should see `received` followed by `refresh_complete` within a few
seconds of the cell edit.
